Welcome to the Improved Hamilton County Health Dashboard Data Portal


  • Introduction
  • CPSC 4910 UTC Spring 2024 Capstone Project - Group 2
  • This web application serves as supporting infrastructure for our health dashboard, holding and managing the data which it consumes
  • It is not meant to be public facing and wasn't the main priority of our project, but it plays an important role and was fun to develop
  • Full CRUD capability is present in this web portal, but most of our mock data was inserted through client-side scripts
  • Our Public Facing Dashboard
  • Data Organization
  • Disease Categories - group diseases at a high-level
  • Diseases - individual diseases which belong to a single category
  • Status Reports - five types, applicable to a single disease during a set time interval
  • Status Reports
  • Case Reports - generic case count and demographic information
  • Hospitalized Reports - applies to hospitalized inpatients, those under observation, and those in the ICU
  • Death Reports - number of deaths attributed to a disease for some interval
  • API
  • Interaction with this portal requires authentication, but we expose a public API for use by our dashboard
  • Access API at https://hcdbackend.fly.dev/dataportal/_query2
  • URL Args: type, disease, disease_category
  • type: disease_category, disease, case_report, death_report, hospitalized_report
  • disease_category: Pass to filter a report type by a disease category (use disease category.name)
  • disease: Pass to filter by a specific disease (use disease.name)
